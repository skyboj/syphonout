## Architecture
- **Rust staticlib** (`core/`): `libsyphonout_core.a` — full state machine, Metal renderer
  (zero-copy IOSurface → MTLTexture via `newTextureWithDescriptor:iosurface:plane:`),
  C FFI boundary (`syphonout_core.h`)
- **SyphonNative.m**: dlopen-based Syphon.framework loader; no compile-time framework dep;
  extracts `IOSurfaceRef` from `SyphonIOSurfaceImage._surface` ivar via ObjC runtime offset;
  manages per-display `SyphonClient` instances with shared `CGLContext`
- **Swift thin layer**: `AppDelegate` boots Rust + SyphonNative; `OutputWindowController`
  creates `CAMetalLayer` + `CVDisplayLink` per display; `StatusBarController` + `MenuBuilder`
  drive the menu bar UI via Rust FFI calls

## Key Technical Notes
- metal 0.29 API: `set_fragment_bytes(index, length, bytes)` — index FIRST
- objc2 ↔ metal interop: cast `device.as_ptr()` → `*mut AnyObject` for `msg_send!`
- IOSurface extraction: `ivar_getOffset` + raw byte arithmetic (not `object_getIvar` — CF type)
- CGLContext required for SyphonClient; shared single context across displays, locked with CGLLockContext
- Syphon.framework x86_64 only → runs under Rosetta; Metal works in same process
- Shaders compiled by Xcode → `default.metallib` → loaded via `device.new_default_library()`

- **Task:** fix(solink): separate render targets from IOSurfaces — OBS crash
- **Status:** Done (commit 8e2d919, pushed to origin/main)
- **Summary:** Fixed OBS crash in normal mode. Root cause:
  gs_texture_create_from_iosurface() returns textures that are NOT valid
  render targets in OBS's OpenGL backend. gs_set_render_target() failed with
  "Texture is not a render target" / "device_set_render_target (GL) failed".

  then copy pixels via gs_stage_texture() → gs_stagesurface_map() → IOSurface.
  Uses OBS's cross-backend API, works on both OpenGL and Metal.
